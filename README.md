# python-algorithm-note

Python algorithm problem-solving notes, solutions, and wrong-answer reviews for coding interview practice.

## Repository conventions

This repository follows the Codex repository conventions and commit message convention defined in [`AGENTS.md`](AGENTS.md).
## Project purpose

`python-algorithm-note` is a personal knowledge base for Python algorithm practice. It is intended to collect problem-solving notes, solution ideas, implementation patterns, and wrong-answer reviews for coding interview and competitive programming preparation.

## Repository structure

This repository is designed to be published as a MkDocs site. A typical layout is:

```text
.
├── README.md                  # Project overview and contributor guidance
├── mkdocs.yml                 # MkDocs configuration
├── docs/                      # Markdown content for the site
│   └── index.md               # Site landing page
└── .github/
    └── workflows/
        └── deploy.yml         # GitHub Actions workflow for deployment
```

As the project grows, add new Markdown files under `docs/` and organize them by topic, platform, or problem category.

## Local setup

Create and activate a virtual environment, then install the MkDocs Material theme:

```bash
python -m venv .venv
source .venv/bin/activate
pip install mkdocs-material
```

## Local preview

Run the local development server:

```bash
mkdocs serve
```

MkDocs will print a local URL that you can open in your browser. The preview server reloads automatically when Markdown or configuration files change.

## Static build

Build the static site locally:

```bash
mkdocs build
```

The generated site is written to the `site/` directory by default.

## Deployment

Pushes to the `main` branch trigger the GitHub Actions workflow at `.github/workflows/deploy.yml`. That workflow is responsible for building and publishing the MkDocs site.

If deploying with `mkdocs gh-deploy`, configure GitHub Pages for this repository to serve from the `gh-pages` branch. This lets GitHub Pages publish the static files generated by MkDocs.

## Content policy

To keep this repository safe and respectful:

- Do not include private information.
- Do not commit API keys, access tokens, passwords, or other secrets.
- Do not copy full copyrighted problem statements into this repository.
- Link to original problems instead of copying statements.
