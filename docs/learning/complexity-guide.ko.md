# 복잡도 가이드

## 왜 시간 복잡도가 중요한가

로직이 맞아도 실행 시간이 길면 실패할 수 있습니다.
입력 크기에 맞는 알고리즘 선택이 필요합니다.

## N 크기 판단 표

| 입력 크기 (N) | 권장 복잡도 |
|---|---|
| 최대 1,000 | O(N^2) 허용 가능 |
| 최대 100,000 | 보통 O(N log N) 또는 O(N) 필요 |
| 최대 1,000,000 | 보통 O(N) 또는 그 이상 효율 필요 |

## 자주 쓰는 복잡도

- O(1): 상수 시간 조회/연산
- O(log N): 이진 탐색
- O(N): 한 번 순회
- O(N log N): 정렬 기반
- O(N^2): 이중 루프
- O(NQ): N과 Q를 함께 도는 구조(대부분 느림)

## 반복 구간 합이 왜 느린가

쿼리마다 `sum(arr[L:R])`를 수행하면 쿼리 1개가 O(N)입니다.
Q개 쿼리를 처리하면 총 O(NQ)까지 커질 수 있어 큰 입력에서
시간 초과가 발생합니다.

## 복잡도 체크리스트

- 먼저 입력 크기를 확인했는가
- 최악 연산 횟수를 추정했는가
- 제약에 맞는 자료구조를 선택했는가
