# 누적합(prefix sum)

## 이 개념은 무엇인가
누적합(prefix sum)은 "앞에서부터 현재 위치까지의 합"을 미리 저장해 두고, 이후 구간 합을 빠르게 계산하는 기법입니다.

## 언제 사용하는가
- 구간 합 질의가 여러 번 들어오는 문제
- 고정된 배열에서 [L, R] 합을 반복 계산해야 하는 문제
- 직접 합산하면 시간복잡도가 커지는 상황

## 핵심 아이디어
- `prefix[0] = 0`으로 시작하면 1-based index 질의를 일관된 공식으로 처리할 수 있습니다.
- 1-based 질의의 핵심 공식은 `prefix[r] - prefix[l - 1]`입니다.
- 질의마다 직접 더하면 `O(NQ)`가 될 수 있지만, 누적합은 전처리 `O(N)` + 질의 `O(1)`로 줄입니다.

## 기본 문법 또는 패턴
```python
prefix = [0] * (n + 1)
for i in range(1, n + 1):
    prefix[i] = prefix[i - 1] + arr[i - 1]

# 1-based query [l, r]
range_sum = prefix[r] - prefix[l - 1]
```

## 단계별 예시
1. `arr = [10, 20, 30, 40, 50]`를 준비합니다.
2. `prefix = [0, 10, 30, 60, 100, 150]`를 만듭니다.
3. `l=2, r=4`라면 `prefix[4] - prefix[1] = 100 - 10 = 90`입니다.
4. 즉 `20 + 30 + 40`을 매번 직접 더하지 않아도 즉시 구간 합을 얻습니다.

## 흔한 실수
- `prefix[0]`을 두지 않아 `l=1` 케이스에서 예외 처리가 늘어나는 실수
- 0-based index와 1-based index를 섞어 off-by-one 오류를 내는 실수
- 질의마다 `sum(arr[l-1:r])`를 호출해 성능을 잃는 실수
- `r`이 포함 구간이라는 조건을 놓치는 실수

## 안전한 패턴
- 문제의 index 체계(0-based/1-based)를 먼저 고정합니다.
- 누적합 배열 길이를 `n+1`로 두고 `prefix[0]=0`을 명시합니다.
- 질의 공식을 한 가지로 통일해 예외 분기를 줄입니다.
- 경계값(`l=1`, `r=n`) 테스트를 먼저 수행합니다.

## 시간복잡도
- 누적합 전처리: `O(N)`
- 각 구간 합 질의: `O(1)`
- `Q`개 질의 전체: `O(N + Q)`
