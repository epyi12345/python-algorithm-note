# 시간복잡도(time complexity)

## 이 개념은 무엇인가
입력 크기 N이 커질 때 연산 횟수가 어떻게 증가하는지 나타내는 기준입니다.

## 언제 사용하는가
문제 풀이 방법을 고르기 전, 제한 시간 내 통과 가능성을 판단할 때 사용합니다.

## 핵심 아이디어
최악 케이스 기준으로 상한을 계산하고, 입력 제한과 비교해 가능한 접근만 남깁니다.

## 기본 문법 또는 패턴
| 표기 | 의미 |
|---|---|
| O(1) | 상수 시간 |
| O(log N) | 절반씩 줄어드는 탐색 |
| O(N) | 1회 선형 순회 |
| O(N log N) | 정렬 + 선형 처리 |
| O(N^2) | 이중 반복 |
| O(NQ) | N과 Q를 곱한 반복 |

## 단계별 예시
1. N, Q 최대값을 확인합니다.
2. 후보 알고리즘의 반복 횟수를 식으로 씁니다.
3. 대략적인 연산량을 계산합니다.
4. 너무 큰 경우(예: O(N^2), O(NQ))는 즉시 제외합니다.

## 흔한 실수
- 평균 케이스만 보고 최악 케이스를 놓치는 경우
- 중첩 반복을 과소평가하는 경우
- 파싱/출력 비용을 무시하는 경우

## 안전한 패턴
- 구현 전에 복잡도 먼저 계산
- 경계값으로 연산량 재점검
- 통과 가능한 상한만 구현

## 시간복잡도
문제마다 다르지만, 보통 N이 100000 수준이면 O(N)~O(N log N)을 우선 검토합니다.
